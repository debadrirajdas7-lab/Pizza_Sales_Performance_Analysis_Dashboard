🍕 Pizza Sales Performance Analysis Dashboard

📌 Project Overview

This project focuses on analyzing pizza sales data to uncover key business insights and trends that can support data-driven decision making for a restaurant business. The analysis examines sales performance, customer ordering patterns, and product popularity.

Using SQL for data analysis and Power BI for visualization, an interactive dashboard was developed to present important key performance indicators (KPIs) and actionable insights.

🎯 Problem Statement

Restaurants generate large volumes of transactional data daily, but without proper analysis it becomes difficult to extract meaningful insights from this data.

The objective of this project is to analyze historical pizza sales data to answer important business questions such as:

• What is the total revenue generated by the restaurant?

• Which pizzas are the best and worst sellers?

• What are the peak order times during the day?

• Which pizza categories and sizes contribute the most to overall sales?

The goal is to transform raw sales data into actionable insights that support strategic decision-making.

📂 Dataset

The dataset used in this project contains detailed transactional records of pizza orders. Each row represents a pizza item included in a customer order.

Key Columns in the Dataset

• order_id – Unique identifier for each order

• order_date – Date when the order was placed

• order_time – Time when the order was placed

• pizza_name – Name of the pizza ordered

• pizza_category – Category of the pizza (Classic, Veggie, Chicken, etc.)

• pizza_size – Size of the pizza (Small, Medium, Large, etc.)

• quantity – Number of pizzas ordered

• total_price – Total price of the pizza order

The dataset provides the necessary information to analyze sales performance across different time periods, pizza categories, and product sizes.

🛠 Tools Used

The following tools and technologies were used in this project:

• SQL – Used for data querying and KPI calculations

• Power BI – Used for building interactive dashboards and visualizations

• Microsoft Excel / CSV – Used for storing and managing the dataset

📊 Key Performance Indicators (KPIs)

To evaluate the performance of pizza sales, the following KPIs were calculated:

• Total Revenue – Total sales generated from pizza orders

• Total Orders – Total number of customer orders

• Total Pizzas Sold – Total quantity of pizzas sold

• Average Order Value – Average revenue generated per order

• Average Pizzas per Order – Average number of pizzas purchased per order

These KPIs help measure overall business performance and customer purchasing behavior.

🗄 SQL Analysis

SQL was used to analyze the dataset and extract meaningful insights.

• Total Revenue

SELECT SUM(total_price) AS Total_Revenue
FROM pizza_sales;

• Total Orders

SELECT COUNT(DISTINCT order_id) AS Total_Orders
FROM pizza_sales;

• Top 5 Best-Selling Pizzas

SELECT TOP 5 pizza_name, SUM(quantity) AS Total_Sold
FROM pizza_sales
GROUP BY pizza_name
ORDER BY Total_Sold DESC;

• Sales by Pizza Category

SELECT pizza_category, SUM(total_price) AS Revenue
FROM pizza_sales
GROUP BY pizza_category
ORDER BY Revenue DESC;

These queries help identify top-performing products, revenue contribution, and customer ordering trends.

🔍 Key Insights

The analysis revealed several important business insights:

• Weekend sales are significantly higher compared to weekdays.

• Large-sized pizzas contribute the highest share of revenue.

• The Classic pizza category generates the highest overall sales.

• A small number of pizza types account for a large percentage of total revenue.

These insights highlight customer preferences and peak sales periods.
